算法概览

  1. 数据结构：

    • 数组
    • 链表
    • 字符串
    • 二叉树

  2. 处理思路：

    • 基本数据结构，配合基本数学运算
    • 高级点的，还需要考虑函数的递归调用模型，类似于高阶函数和数列
    • 涉及循环或递归这些重复计算都需要考虑退出条件
    • 算法的效率由时间复杂度和空间复杂度共同决定

  3. 数值运算：

    • 最基本的是加法和减法，然后是乘法和除法
    • 取模运算和整数除法
    • 乘法和加法会造成数值溢出
    • 最大值和最小值经常用于动态规划
    • 位运算主要涉及移位操作，有左移和右移
    • 不同的数据类型的精度不同
    • 逻辑运算有与或非和异或

  4. 经典算法：

    • 暴力求解
    • 二分查找
    • 遍历
    • 双指针
    • 递归
    • 回溯
    • 动态规划
    • 分治
    • 哈希
    • 归并
    • 位图
    • AVL树
    • KMP算法
    • Manacher算法

  5. 回顾经典数学思想：

    • 函数与方程
    • 分类与分解
    • 数形结合
    • 归纳与演绎
    • 特殊到一般

暴力求解

不计较算法的开销，采用直观的方式进行求解，枚举可能出现的每一种情况。该方法虽然执行效率低，但相对直观，有助于发现冗余，打开思路。需要注意的是，为了获得正确的结果，需要着重考虑不同的输入情形下对应的边界条件，防止溢出。

二分查找

注意mid = low + (high - low)/2, 中间数一定位于low和high之间，不能用high+low计算，加起来的值会溢出。

int binarySearch(int[] A, int target) {
   int low = 0, high = A.length - 1;
   int mid;
   while(low <= high) {
       mid = low + (high - low)/2; 
       if(A[mid] == target) {
          return mid;
       }else if(A[mid] > target){
           high = mid - 1;
       }else {
           low = mid + 1;
       }
   }
   return -1;
}
  1. 举例：A = [0,1,2,3,4,5]
    • 初始化：low=0,high=5,mid=0,target=10;
    • 1) mid=2, A[mid] < target;
    • 2) low=3,high=5,mid=4, A[mid] < target;
    • 3) low=5,high=5,mid=5,A[mid] < target;
    • 4) low=6,high=5,退出循环 return -1;

遍历

遍历：按照一定规则依次访问每一个元素。

  1. 数组遍历：最典型的场景是数组进行反转
  2. 链表遍历：配合双指针可以近似实现随机访问节点的目的
  3. 二叉树遍历：通过递归的方式访问每一个节点

双指针

双指针：在循环遍历链表过程中通过快慢指针来记录当前访问的节点。

  1. 适用场景：
    • 访问链表的倒数第K个节点
    • 访问链表的中间节点
    • 检测单链表中是否有环

递归

递归：一类通用算法的代名词，本质是基于编程语言的函数的栈式调用结构。

  1. 递归的陷阱

    • 效率低，调用栈太深
    • 容易溢出，边界条件不好控制

  2. 递归的三大要素

    • 函数定义：明确函数的输入输出和功能定位，越简单越好
    • 退出条件：发现递归结束条件，从最简单或最初始的状态入手
    • 递归方程：自顶向下，发现递归式，从通用的范式入手
      发现递归条件：可以理解为，从f(1)…正向推到f(n),从初始到无穷，从简单到一般。
      发现递归式：即发现问题的本质，从通用着手，从f(n)…反推到f(1).

  3. 场景：

    • 求阶乘，斐波那契数列
    • 反转单链表
    • 二叉树遍历

  4. 递归的优化

    起因：子问题被重复计算。

    • 备忘录：保存递归过程的中间计算结果
    • 考虑是否可以自底向上的递推

回溯

回溯：back tracking，属于路径搜索算法中的一种，通过不断的去试错，控制路径的前进和后退来选取最终的目标。

  1. 经典问题

    • 01背包问题
    • 8皇后问题

  2. 回溯的陷阱

    • 可看作深度优先搜索
    • 全搜索，复杂度高

  3. 设计思路

res = [];       全局变量 保存最终的结果
state = [];     状态变量 保存当前的状态
p,q,r;          条件变量 决定搜索是否满足条件

back(状态，条件1, 条件2, ...) {
    if(不满足条件则剪枝操作) {
        return;
    }else { 状态满足最终的要求
        res.add(state);
        return;
    }

    // 遍历每一个阶段的可选解
    for 依次遍历和递归 {
        if(满足执行条件) {
            已选解集合.add(可选解)
            back(状态，条件1, 条件2, ...);
            已选解集合.remove(可选解)
        }
    }
}
最终从res中获取结果
  1. 适用场景
    • 排列、组合（子集、幂集、字符全排列）。 在传值时，对于排列问题，是要删掉单个用过的元素；组合问题，是删掉前面所有的元素。
    • 数组、字符串，给定一个特定的规则，尝试搜索迭代找到某个解。
    • 二维数组下的DFS搜索（八皇后、黄金矿工、数独）

动态规划

自下向上，从简单到复杂，从特殊到一般。

  1. 基本属性
    • 最优子结构
    • 重叠子问题
  2. 突破点
    • 状态定义
    • 状态迁移
